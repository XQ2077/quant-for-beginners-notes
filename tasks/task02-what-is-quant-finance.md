## 3. 知识点总结

### 3.1 量化金融的核心

量化金融是用数据和明确规则辅助金融决策，而不是依靠感觉猜测下一次涨跌。一个想法只有被写成可以计算的指标、用历史数据检验，并转化为可重复执行的规则，才进入量化研究范畴。课程给出的核心循环是：

```text
提出假设 → 获取数据 → 历史验证 → 形成规则 → 持续迭代
```

### 3.2 从研究到交易的基本流程

完整流程通常包括数据获取、清洗与分析、策略设计、历史回测和模拟交易。数据分析回答市场过去发生了什么，策略规则说明在什么条件下行动，回测检验规则在历史上的收益和风险，模拟交易再验证数据延迟、成交和程序运行等实际问题。Python 为这些环节提供统一工具链，因此适合快速把研究想法转成可运行实验。

### 3.3 随机性与统计规律

随机游走实验说明，单条价格路径可能受到大量随机波动影响，很难准确预测每一步；但重复模拟许多路径后，最终价格的分布、均值和离散程度会呈现统计规律。量化研究的重点不是消除不确定性，而是在承认不确定性的前提下，寻找可以重复验证的概率优势并控制风险。

### 3.4 概率优势、仓位与长期纪律

较高胜率并不自动等于长期盈利。最终结果同时取决于胜率、盈亏比、每次投入比例和能否持续执行。如果仓位过大，少量连续亏损就可能严重破坏本金，使后续即使出现盈利机会也难以恢复。因此，量化策略需要把“有没有优势”和“为这个优势投入多少资金”分开研究。

### 3.5 长期平均对数增长率与凯利公式

多次交易后的财富按复利连乘。假设胜率为 $p$、败率为 $q=1-p$，每投入 1 元在盈利时净赚 $b$ 元，每次投入资金比例为 $f$，则每次交易的期望对数增长率为：

$$
g(f)=p\ln(1+bf)+q\ln(1-f)
$$

对 $f$ 求最大值，可以得到凯利仓位：

$$
f^*=\frac{bp-q}{b}
$$

对数把财富的连乘转换为相加，并会严厉惩罚接近破产的结果。以课程中的 1:1 盈亏比和 52% 胜率为例，凯利仓位为 $2\times52\%-1=4\%$：仓位更小会降低理论增长速度，仓位过大则可能使长期对数增长率转为负数。

### 3.6 凯利公式的适用限制

凯利公式只在“最大化长期复利增长率”这一目标下最优，而且依赖胜率、盈亏比相对稳定、交易可以重复等假设。真实市场还存在估计误差、相关性、手续费、滑点和大幅回撤，因此满凯利通常过于激进，半凯利或四分之一凯利更能容忍参数估计偏差。AI 和数据工具可以辅助研究，但不能替代对策略逻辑、数据质量和风险的判断。

### 3.7 关键函数与量化实验算法

#### 函数与方法速查

| 函数或方法 | 关键参数 | 返回值 | 本 Task 中的用途 |
| --- | --- | --- | --- |
| `yf.download()` | `tickers`、`period`、`progress`、`multi_level_index` | 以日期为索引的行情 `DataFrame` | 下载 TSLA 最近 6 个月的 OHLCV 日线数据 |
| `len(obj)` | 任意支持长度协议的对象 | 整数 | 统计行情表中的交易日数量 |
| `DataFrame.tail(n)` | `n` 为末尾行数，默认 5 | 原表末尾 $n$ 行组成的新表 | 快速检查最新几行数据和日期是否正常 |
| `DataFrame.iloc[index]` | 按位置取行或元素；`-1` 表示最后一个 | 标量、`Series` 或 `DataFrame` | 获取最新一个交易日的收盘价 |
| `plt.subplots()` | 子图行列数、`figsize`、`sharex` 等 | `Figure` 和一个或一组 `Axes` | 创建价格与成交量上下排列的画布 |
| `Axes.plot()` | 横轴、纵轴、颜色、线宽等 | `Line2D` 对象列表 | 绘制连续的收盘价时间序列 |
| `Axes.bar()` | 横轴位置、柱高、宽度和颜色等 | `BarContainer` | 绘制每天的成交量柱状图 |
| `np.random.normal()` | 均值 `loc`、标准差 `scale`、样本形状 `size` | 随机样本数组 | 生成随机游走或蒙特卡洛实验中的日收益率 |
| `np.cumprod()` | 输入数组和累计轴 `axis` | 累积乘积数组 | 把每日增长因子连乘为价格路径或资金曲线 |

`period='6mo'` 表示相对运行日期向前滚动 6 个月，并不是固定起止日期。`multi_level_index=False` 让单个标的返回普通列索引，便于直接使用 `data['Close']`；如果同时下载多个标的，列结构和取值方式会发生变化。

#### 行情与量化实验流程

1. **行情获取与验证**：用 `yf.download()` 获取数据，先检查 `data.empty`，再访问最后一行，避免空表触发索引错误。
2. **时间序列展示**：价格使用折线图保留连续变化，成交量使用柱状图强调每日规模；两个子图共享日期轴。
3. **随机游走**：从正态分布抽取日收益率 $r_t$，通过 $P_t=P_0\prod_{i=1}^{t}(1+r_i)$ 生成一条价格路径。
4. **重复模拟**：生成 $S$ 条、每条 $T$ 期的路径，比较终点资金分布，而不是只解释某一条偶然路径。
5. **凯利仓位**：根据胜率 $p$、败率 $q=1-p$ 和净赔率 $b$ 计算 $f^*=(bp-q)/b$，再比较满凯利、低仓位和过度重仓的长期复利差异。

#### 复杂度与边界情况

下载后的单条行情包含 $n$ 个交易日，检查、绘图和 `cumprod()` 都需要顺序处理数据，时间复杂度为 $O(n)$，保存数据的空间复杂度也是 $O(n)$。若蒙特卡洛实验包含 $S$ 条路径、每条 $T$ 轮，时间复杂度为 $O(S\times T)$；保存全部资金轨迹时空间复杂度同样为 $O(S\times T)$，只保留终点值则可降为 $O(S)$。

- 网络失败、代码错误或标的退市都可能得到空表；对空表执行 `iloc[-1]` 会抛出 `IndexError`。
- 滚动时间窗口会随运行日期变化，因此截图中的交易日数量和最新价格不是永久固定值。
- `np.random.normal()` 生成的是模型假设下的随机数，不代表真实收益一定服从正态分布；固定随机种子只能保证模拟可重复。
- 凯利仓位对胜率和赔率估计非常敏感；当估计误差较大、收益相关或存在交易成本时，理论最优值可能过于激进。

#### 最小示例

```python
import numpy as np
import yfinance as yf

data = yf.download(
    'TSLA', period='6mo', progress=False, multi_level_index=False
)
if data.empty:
    raise RuntimeError('未获取到 TSLA 行情')

latest_close = data['Close'].iloc[-1]
print('交易日：', len(data), '最新收盘价：', round(float(latest_close), 2))

# 用随机日收益率生成一条复利价格路径
rng = np.random.default_rng(2026)
daily_returns = rng.normal(loc=0.0008, scale=0.02, size=120)
price_path = 100 * np.cumprod(1 + daily_returns)

# 1:1 赔率、52% 胜率下的凯利仓位
p = 0.52
b = 1.0
q = 1 - p
kelly_fraction = (b * p - q) / b
print('路径终值：', round(float(price_path[-1]), 2))
print('凯利仓位：', f'{kelly_fraction:.2%}')
```
